# Анализ disb (дисбаланса агрессивных сделок) как предиктора движения цены

## Данные
- **Таблица:** `moex.tradestats_fo` на ClickHouse `10.0.0.63:8123`
- **Колонки:** `secid`, `tradedate`, `tradetime`, `pr_open`, `pr_high`, `pr_low`, `pr_close`, `disb`, `vol_b`, `vol_s`, `vol`, `oi_close`
- **Период:** 2020-01-03 до 2026-06-18
- **Тикеры:** все фьючерсы MOEX (Si, HY, GL, HS, RN, NM и др.)

## Что проверить

### 1. Корреляция disb → next_bar_return (1-минутные бары)
- `next_return = (pr_close.shift(-1) - pr_close) / pr_close`
- `current_disb = disb`
- Пирсон/Spearman корреляция на всём периоде
- На каких тикерах корреляция значима (p < 0.05)?

### 2. Ресемплинг на 5-минутки (как в Super Candles)
- Агрегировать disb: среднее, сумма, последнее значение за 5 мин
- Агрегировать цены: OHLC за 5 мин
- Посчитать next_return для 5-минутных баров
- Сравнить с 1-минутной корреляцией

### 3. Простая стратегия на disb
- Сигнал: `disb > threshold` → long, `disb < -threshold` → short
- `threshold` от 0.3 до 0.9 с шагом 0.1
- Держим 1 бар (1 мин)
- Комиссия: 4 руб/контракт
- Расчёт: total_return, Sharpe, win_rate, max_drawdown

### 4. Сравнение с oi_close (изменение OI за 1 мин)
- `oi_diff = oi_close - oi_open`
- Корреляция oi_diff → next_return
- Комбинация disb + oi_diff (линейная регрессия)

### 5. Важно!
- **Look-ahead bias:** все расчёты только с `.shift(1)` — никаких данных текущего бара в сигнале
- **Только внутридневные данные** (10:00-18:45 МСК)
- Фильтр: тикеры с объёмом > 100 контрактов/день
