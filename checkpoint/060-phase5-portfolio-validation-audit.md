### Все баги (по порядку)

| # | Проблема | Причина | Статус |
|:-:|----------|---------|:------:|
| 1 | **NameError: n** в `min.log` | Переменная `n` не определена, должно быть `len(mpnl)` | Исправлено |
| 2 | **0 trades, 0 сделок** в `final_clean.json` | Timezone mismatch: ClickHouse отдаёт timezone-aware timestamps (Asia/Irkutsk +08:00), а `all_ts` — naive. `ts in sc_arr.index` не находит совпадений | **НЕ ИСПРАВЛЕНО** |
| 3 | **timeout 143** (SIGTERM) на 60K+ баров | Слишком долго — нужна оптимизация | Работает через оригинальные скрипты |
| 4 | **exit code -15** (SIGTERM) на shell-обёртке | То же — таймаут bash | — |

### Корень проблемы
- ClickHouse отдаёт **timezone-aware timestamps** (`pd.Timestamp('2025-01-15 10:00:00+08:00')`)
- `all_ts = sorted({t for df in data.values() for t in df.index})` объединяет их в naive (после `.tz_localize(None)`)
- При `ts in sc_arr.index` — pandas не может найти naive timestamp в timezone-aware индексе Series
- Оригинальные скрипты Phase 5 используют свой `SignalCache` и другую логику итерации, без этой проблемы

## План аудита (что нужно сделать дальше)

### Задача 1: Исправить верификацию
- **Вариант A:** При загрузке из ClickHouse делать `.dt.tz_localize(None)` на всё сразу — до построения `all_ts`
- **Вариант B:** Использовать `pd.Timestamp(ts).tz_localize('Asia/Irkutsk')` чтобы ts совпадал с индексом
- **Вариант C:** Не писать свой велосипед — написать самодостаточный скрипт, который импортирует `SigGen` из `trading_bot/new_strategies.py` и запускает ту же симуляцию что и оригинал, но сохраняет monthly PnL

### Задача 2: Аудит стратегии (критично!)
Проверить на **look-ahead bias** и **overfitting**:

1. **Ловля тренда (trend-following bias)**
   - Сравнить PnL по периодам: бычий (2023-2024) vs боковой (2025-2026)
   - Если доходность резко падает в боковике — стратегия ловит тренд, а не alpha
   - Проверить correlation с индексом MOEX (IMOEXF, MIX)

2. **Подгонка состава портфеля по OOS**
   - В walk-forward: проверь, какие тикеры отбираются в каждом окне
   - Если один и тот же набор тикеров проходит каждый раз → портфель стабилен
   - Если каждый раз разные → отбор переобучен

3. **Проверка случайных сделок вручную**
   - Взять 5-10 случайных сделок из лога
   - Вручную воспроизвести entry/exit по барам
   - Проверить: entry_time, entry_price, exit_time, exit_price, direction, PnL

4. **Проверка комиссий**
   - Включают ли скрипты комиссии MOEX?
   - Какая комиссия заложена? (обычно 0.01% + ГО 12-20%)

5. **Monte Carlo shuffle**
   - Перемешать сигналы → посмотреть distribution возврата
   - Если 95% результатов хуже unshuffled → стратегия несёт информацию
   - Если нет → data snooping

6. **Out-of-sample на неотобранных тикерах**
   - Запустить стратегию с теми же параметрами на бумагах, которые НЕ вошли в портфель
   - Если сливает → тикеры подобраны под стратегию, а не стратегия работает
