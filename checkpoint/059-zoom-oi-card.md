# 059 — Zoom-карточка с OI на дашборде Phase2

## Контекст/проблема

На дашборде MOEX OI Dashboard (порт 8086) не было возможности посмотреть сделку вблизи. Клик по графику ничего не делал. Требовалось:

1. Zoom-карточка сделки под графиком цены (canvas с ценой + OI линиями)
2. Клик по графику → zoom ближайшей Phase2 сделки
3. Номер сделки возле треугольника входа
4. Читаемые шрифты на осях
5. Данные OI (fiz_long, fiz_short, yur_long, yur_short) на zoom-графике

## Методология

- Всё реализовано на клиентской стороне (чистый JS в HTML-шаблоне)
- Серверная часть (Python) не менялась, кроме фикса бага `np.where` -> `np.array`
- Zoom использует уже загруженные `_lastData` — не перезагружает данные
- OI линии рисуются по второй шкале (правая ось)

## Результаты

### Что реализовано

1. **HTML-структура** (строка 367): zoomCard вставлена внутрь card-full с c0 (сразу под графиком цены), до сетки из 4 карточек
2. **Клик по c0** (строка 694): поиск ближайшей Phase2 сделки по индексу бара → `showZoom()`
3. **showZoom()** (строка 807):
   - Показывает zoomCard (display: block)
   - Рисует на canvas #cz приближение сделки: 60 баров до входа, 40 после выхода
   - Цена — синяя линия, левая шкала
   - OI линии: fiz_long (зелёный), fiz_short (красный), yur_long (голубой пунктир), yur_short (жёлтый пунктир) — правая шкала
   - Легенда OI в правом верхнем углу
   - Зелёная пунктирная вертикальная линия ENTRY
   - Красная пунктирная вертикальная линия EXIT
   - Горизонтальные пунктиры от entry/exit цены до оси
   - Стрелка ENTRY→EXIT оранжевая/красная
   - Номер сделки белым текстом справа от треугольника (#N)
   - Инфо-текст под canvas: тикер, entry/exit время, цена, PnL, бары
   - Шрифты: оси 16px bold, ENTRY/EXIT 16px bold, info 14px

### Фиксы багов

- **np.where bug** (строка 58): `toi = np.where(..., 1, 1)` → `toi = np.array([float(r[10]) if float(r[10]) > 0 else 1 ...])` — из-за бага `dtoi` всегда было 0, все паттерны с `dtoi>0` не срабатывали
