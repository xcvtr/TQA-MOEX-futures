# 036 — Symmetrical LONG+SHORT Signals: Balanced Score (2026-06-15)

## Контекст
Текущая стратегия BASE v2 работала только в LONG (score>0.10). Рынок даёт движения в обе стороны — необходимо добавить SHORT-сигналы для симметричного извлечения прибыли.

## Рассмотренные варианты

### 1. Зеркальный SHORT (отброшен)
- Простое отражение: direction='S' при score>0.10
- Проблема: score считался только для LONG (vs обрезан в [0,1])
- **Отброшен** — не использует данные рынка для SHORT

### 2. Асимметричный score_raw (отброшен)
- `vs_raw = (vr - 1.5) / 3.0` в [-1, 1]
- Давал **64.8% SHORT vs 22.1% LONG** — сильный перекос
- Причина: объём MOEX фьючерсов систематически ниже среднего (медиана vr≈1.0, порог 1.5)
- **Отброшен** — несимметричный

### 3. vz вместо vr для vs (тестировался)
- `vs = clip(vz / 2, -1, 1)` — z-score объёма
- 29.2% LONG vs 52.0% SHORT — всё ещё skewed
- **Отброшен** — OI-компонента доминирует

### 4. Сбалансированный: tanh(vz/3)*0.3 + os_*0.7 (ВЫБРАН ✅)
- `vs = tanh(vz / 3)` — симметричный [-1, 1]
- `os_ = clip((oima - oi_r) / oima, -1, 1)` — симметричный OI-компонент
- Веса: vs=0.3, os_=0.7 — OI доминирует (он симметричнее)
- **21.4% LONG vs 20.1% SHORT** — почти идеальный баланс
- **Выбран** — наиболее сбалансированное распределение

## Методология
- Score: `score_sym = tanh(vz/3)*0.3 + clip((oima - oi_r)/oima, -1, 1)*0.7`
- Adjust: `* clip(1 - (atr_pct - 0.3) / 3.0, 0, 1)` (volatility filter)
- Порог: `abs(score_sym) > 0.10`, направление по знаку
- Параметры: score_thresh=0.10, bars=8, stop=1.0ATR, lot=100% (HY/AF=75%)
- Период: 2024-01-01 — 2026-05-01
- Портфель: GL, HS, HY, RN, NM, AF

## Ключевые решения с обоснованием

1. **Почему vs через tanh(vz/3), а не (vr-1.5)/3.0** — vr имеет медиану ~1.0, порог 1.5 даёт 70% SHORT. vz (z-score) уже симметричен вокруг 0. tanh сглаживает выбросы без clamp-искажений.

2. **Почему веса 0.3/0.7, а не 0.5/0.5** — os_ (OI-компонента) симметричнее (13.6% LONG, 18.2% SHORT), vs (объём) склонён к SHORT. Уменьшение веса vs до 0.3 даёт почти идеальный баланс 21/20%.

3. **Почему lot=100% для всех (HY/AF=75%)** — предыдущий leverage sweep (checkpoint 034) показал что с SHORT DD почти не растёт, можно держать 100%.

## Что не сработало и почему
- **Kelly sizing** (checkpoint 033): Kelly при WR=35% и payoff <1.5 даёт f* <0.05 — позиция слишком мала. Фиксированный lot лучше.
- **Walk-forward** (checkpoint 032): адаптивные параметры проигрывают фиксированным — рынок стабильнее чем мы думали.
- **ADX/TOD/RAW_OI фильтры** (checkpoint 031): все проиграли BASE score. Score сам достаточно хорош.
