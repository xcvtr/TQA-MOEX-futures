# 037 — LONG+SHORT Production: 6/6 тикеров, Calmar ×10 (2026-06-15)

## Контекст
Задача — добавить SHORT-сигналы в BASE v2. После цепочки исследований (checkpoints 031-036) финальный честный тест на проверенном симуляторе без багов.

## Ключевые решения с обоснованием

1. **Score для SHORT — tanh(vz/3)*0.3 + os_*0.7** — симметричный [-1,1], сбалансирован 21/20%
2. **Порог |score| > 0.10** — одинаков для LONG и SHORT
3. **Стоп 1 ATR** — для SHORT и LONG одинаковый (жёсткий, но с учётом направления)
4. **bars_left = 8** — меньше чем у LONG (13), так как SHORT чаще стопится
5. **Без accounts confidence** — чище сигнал, не искажается

## Что не сработало
- **Accounts confidence** — даёт шум, а не качество (отключено)
- **Walk-forward** — фиксированные параметры стабильнее
- **Kelly sizing** — WR~35-40% даёт f* < 0.05, позиция слишком мала
- **Lot=100%** — contracts=20 при cash=100K, стоп уводит в минус
