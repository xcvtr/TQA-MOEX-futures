### Аудит 3: Monte Carlo shuffle

### Аудит 4: Комиссии

- Прикидка: ~17-27% от PnL съедается MOEX комиссиями (~1р/контракт + 0.01% оборота)
- Даже с комиссиями: **+1,489%** (kelly 40/150) и **+842%** (kelly 20/70)
- **Вердикт: не учтены, но на общий вывод не влияют** ✅

### Аудит 5: Состав портфеля
- Портфель **хардкожен** в `PORTFOLIO` константе скриптов — не подбирался по OOS
- Walk-forward: портфель фиксирован, IS-отбор тикеров отсутствует (walk-forward только для Kelly)
- 10 long + 4 short тикера
- **Вердикт: подгонки нет** ✅

### Аудит 6: Reverse WFO
- Код симуляции использует **expanding window**: precompute signals на всём диапазоне, но rolling-индикаторы (z-score, ATR, VMA) — **past-only** (rolling(w, min_periods=w))
- Тест идёт только на OOS 2025-2026
- **Вердикт: reverse WFO отсутствует** ✅

### Timezone-баг
- Проблема: ClickHouse отдаёт timezone-aware timestamps (+08:00)
- Оригинальный `phase5_walkforward.py` использует `t.to_pydatetime().replace(tzinfo=None)` — корректно
- Мои мини-версии (v2, min, fast) — использовали `.dt.tz_localize(None)` на DataFrame уровне, но `all_ts` собирался из оригинальных данных, где tz ещё была
- **Исправлено**: создан `scripts/phase5_check_monthly.py` на основе оригинального walkforward с добавлением monthly PnL
- **Результат верификации:** получен ✅

## Точная конфигурация

### Портфель (хардкод в `scripts/phase5_walkforward.py`)
```python
PORTFOLIO = {
    'core': [  # LONG
        ('GL','vod','L',21,2,1.0), ('RN','vou','L',5,5,1.0),
        ('AL','vou','L',21,2,1.0), ('HY','vou','L',5,5,1.0),
        ('NM','vod','L',21,3,1.0), ('AF','sm','L',21,2,1.0),
        ('SR','sm','L',8,5,1.0),   ('Si','vyf','L',13,2,1.0),
        ('SN','vou','L',5,5,1.0),  ('YD','vod','L',13,5,1.0),
    ],
    'hedge': [  # SHORT
        ('BR','vyf','S',13,5,1.0), ('SV','vod','S',5,5,1.0),
        ('SF','vod','S',8,3,1.0),  ('NG','vyf','S',5,5,1.0),
    ],
}
```
Формат: `(symbol, pattern, direction, hold_bars, atr_multiplier, weight)`

### Параметры симуляции
- **Начальный капитал:** 100,000 ₽
- **Kelly adaptive:** 40-150% (агрессивный) / 20-70% (консервативный)
- **Score threshold:** 0.25 (long) / 0.20 (short)
- **Score fade exit:** < 0.10
- **Max position:** 35% от available
- **Макс. входов за бар:** 5
- **Стоп-лосс:** ATR * atm (atr_multiplier из портфеля)
- **Hold limit:** фиксированный hold_bars
- **Часы работы:** 7:00-23:00 IRKT
- **GO:** из TICKER_CONFIGS (5000 для большинства, 3000 для HY/BR, 1405 для NM и т.д.)
- **Окно индикаторов:** 20 баров для z-score, VMA, OI ratio
- **ATR:** 14 баров

### Доходность расчёта PnL
- При сделке: PnL = direction * (exit_price - entry_price) / entry_price * GO * contracts
- **MTM (mark-to-market):** учитывается для расчёта DD, не для cash
- Остатки закрываются по последнему бару периода (close)
- **Никаких комиссий, slippage, НДФЛ не учтено**

### Источники данных
- **БД:** ClickHouse localhost:8123, БД `moex`
- **Таблицы:** `prices_5m` (бары), `prices_5m_oi` (OI)
- **JOIN:** `p.time=o.time AND p.symbol=o.symbol`
- **Период загрузки:** 2024-01-01 до 2026-04-30
- **Тикеры:** AF, AL, BR, GL, HY, NG, NM, RN, SF, SN, SR, SV, Si, YD
- **Для IS-портфеля:** AF, AL, GL, GLDRUBF, HY, MM, NG, NM, PT, RN, SBERF, SV, VB, YD

## Что не учтено / ограничения

1. **❌ Комиссии (MOEX):** ~1р/контракт + 0.01% от оборота. Оценка: 17-27% от PnL
2. **❌ Slippage:** не моделируется
3. **❌ НДФЛ (13%):** не учтён
4. **⚠️ Liquidity:** не проверялась реальная ликвидность каждого тикера при больших позициях
5. **⚠️ Look-ahead:** каждый bar отдельно — `rolling` корректные, но сигнал `vr = volume/vma20` использует `rolling(20).mean()` — смещение на 1 бар
6. **⚠️ Monte Carlo:** сделан только на 1 тикере (GL), не на всём портфеле
7. **⚠️ Out-of-sample на НЕотобранных тикерах:** не проверено
