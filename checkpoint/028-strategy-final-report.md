# 028 — Стратегический аудит: ПРАВДА о всех стратегиях

## Контекст
Все старые тесты через `portfolio.py` врали — использовали `exit_price` сигнала как цену закрытия, игнорируя промежуточные стопы. **Bar-level симуляция** (`bar_level_sim.BarLevelPortfolio`) показывает правду: OHLCV close на каждом шаге + mark-to-market + stop-loss на реальных ценах.

**Цель:** найти стратегию с ≥80% годовых при DD ≤ 20% через честную симуляцию.

## Методология
1. **Сигналы:** сгенерированы через функции `trading_bot/new_strategies.py` по всем 47 тикерам (OHLCV 5m, 365 дней)
2. **Обогащение:** каждый сигнал получил score (0..1), atr_pct, adx_value
3. **Фильтр:** score ≥ 0.3 (как в production)
4. **Симуляция:** `BarLevelPortfolio.run()` — OHLCV close, MTM, stop-loss, time-stop, rollover
5. **Параметры портфеля:** mu=0.10, mc=5, tm=0.15, sl=0.01, max_hold=40, use_score_sizing=True, use_score_eviction=True, atr_stop_mult=2.0
6. **Walk-forward:** 256 комбинаций × 4 folds на лучшей стратегии

## TRIZ-анализ

### Противоречие
Стратегия должна давать 80%+ годовых при DD ≤ 20%, но:
- Сигналы не предсказывают будущую цену (типичный возврат: +0.5%..+3.5%)
- Stop-loss на реальных ценах срабатывает чаще, чем в тестах с `exit_price`
- Rollover (замена старой позиции новой) доминирует — 85%+ всех выходов

### Корень проблемы
**Сигналы — шум.** Все 7 протестированных стратегий генерируют сигналы, неотличимые от случайных. Доказательства:
1. Walk-forward: fold 3 всегда отрицателен (−77.4%) независимо от параметров
2. Rollover доминирует — старые позиции не успевают заработать, как приходят новые
3. Wide stops (5%) не улучшают результат — проблема не в жёсткости стопа, а в отсутствии edge
4. Ensemble не помогает — сумма нулей = ноль

### Почему старые тесты показывали +692%?
`portfolio.py` использовал `exit_price` из сигнала как цену закрытия. Это эквивалентно:
- "Я знаю будущую цену через 12 баров, поэтому моя сделка закрылась по этой цене"
- Реальность: между entry и exit стоп-лосс срабатывает на 99% сделок, и только 1% доходит до exit

## Вывод

**НИ ОДНА стратегия не даёт 80% годовых при DD ≤ 20% через честную симуляцию.**

Лучший результат: **OI Divergence Limit — +3.51% return, 24.46% DD** — это шум, а не edge.

Walk-forward подтверждает: результат нестабилен во времени и определяется 1-2 экстремальными сделками.

## Рекомендации

### 1. Сменить парадигму
Текущие стратегии (z-score, OI divergence, mean reversion, VWAP) — **не работают** на MOEX фьючерсах 5m. Нужен принципиально другой подход:

- **Межрыночный анализ:** корреляция Si/BR, RI/GL, spreads
- **Фундаментальные факторы:** ставка ЦБ, нефть, санкции (таймфрейм: дни/недели)
- **Альфа-факторы:** carry trade, contango/backwardation на фьючерсах
- **Machine Learning:** gradient boosting на десятках признаков (не 1-2 z-score)
- **Market Making:** ликвидность, спред, стаканы (не 5m OHLCV)

### 2. Понимать ограничения
- 5m OHLCV фьючерсов **не содержит edge** для простых стратегий
- Любая стратегия с Calmar < 1.0 — это шум
- Если нет стабильности по folds — стратегия не работает

### 3. Что делать с существующим кодом
- `cron_scanner.py` — можно оставить для обучения, но НЕ использовать с реальными деньгами
- Все backtest-результаты >5% — артефакт, а не edge
- `BarLevelPortfolio` — правильный инструмент для честной оценки
