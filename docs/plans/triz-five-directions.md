# План: 5 параллельных направлений TRIZ-поиска edge на MOEX

ИСПОЛЬЗУЙ TRIZ КАЖДЫЙ РАЗ:
- Противоречие → ИКР → Принципы (проскок, асимметрия, динамичность, матрёшка, дробление)
- Data-first: каждое утверждение подкреплено цифрами
- No look-ahead bias
- Bar-level sim для верификации

## Направление 1: Межрыночный анализ
ИКР: Найти пару инструментов, где расхождение даёт предсказуемый возврат.
- Загрузить OHLCV для Si, BR, RI, GL, ED, FF, AU, CNYRUBF (8 ключевых)
- Посчитать rolling correlation (60 periods) между каждой парой
- Найти периоды расхождения > 2σ от средней корреляции
- Проверить: если Si/BR расходятся → BR возвращается через N баров?
- Сделать скрипт cross_market_analysis.py
- Если есть edge → detect_cross_market_signals() + тест через BarLevelPortfolio

## Направление 2: Фьючерсная кривая (contango/backwardation)
ИКР: Структура фьючерсной кривой предсказывает направление цены.
- Найти в БД данные по разным месяцам (Si-9.25 vs Si-12.25, BR-9.25 vs BR-12.25 и т.д.)
- Если данных нет в БД → попробовать загрузить через load_ohlcv с другим suffix
- Посчитать rolling basis: (front_month - back_month) / front_month
- Проверить: если contango > X% → цена растёт/падает?
- Скрипт: futures_curve_analysis.py

## Направление 3: Фундаментальные факторы (daily)
ИКР: Макро-факторы задают направление на days/weeks.
- Загрузить дневные OHLCV для Si, BR, RI, IMOEXF
- Посчитать weekly return
- Проверить корреляцию с: MOEX index, RTS index, USD/RUB
- Проверить: есть ли momentum на weekly timeframe?
- Простая стратегия: если weekly return > 3% → разворот на следующей неделе?
- Скрипт: macro_fundamental_analysis.py

## Направление 4: ML-подход (градиентный бустинг)
ИКР: Комбинация десятков признаков даёт предсказательную силу там, где 1-2 признака — шум.
- Использовать сигналы OI Divergence (уже есть в кэше)
- Для каждого сигнала добавить признаки:
  - z-score цены от 20/50/100 MA
  - ATR(14)/ATR(20)
  - ADX(14)
  - Объём: volume/SMA_volume(20)
  - OI: OI/SMA_OI(20)
  - День недели, месяц, час
  - Возврат за последние 5/10/20 баров
- Целевая: return_pct сигнала > 0 (бинарная классификация)
- Модель: GradientBoostingClassifier (sklearn, n_estimators=100, max_depth=3)
- Train/test split: первые 70% времени train, последние 30% test
- Метрики: accuracy, precision, recall, AUC
- Если AUC > 0.55 → есть edge, сделать стратегию
- Скрипт: ml_edge_detection.py

## Направление 5: DOM / Стакан (если данные есть)
ИКР: Плотность лимитных ордеров предсказывает направление следующей свечи.
- Проверить: есть ли DOM данные в БД (dom_orders, dom_positions таблицы?)
- Если есть: загрузить для Si, BR, AU
- Посчитать: bid/ask imbalance, cluster volume, wall distance
- Проверить: imbalance > 2 → направление следующей свечи совпадает с imbalance?
- Простая стратегия: entry по imbalance, exit по return или reverse imbalance
- Скрипт: dom_analysis.py

## ФОРМАТ ВЫВОДА
Каждый скрипт должен:
1. Сохранить результаты в reports/YYYY-MM-DD-direction-name.md
2. Вывести ТРИЗ-диаграмму: Противоречие → ИКР → Решение
3. Если найден edge → detect_signals + test через BarLevelPortfolio
4. Если edge не найден → объяснить почему

## ТРЕБОВАНИЯ
- Каждый скрипт самодостаточен (импорты внутри)
- Использовать существующую БД (host=10.0.0.64, db=moex)
- Время выполнения каждого скрипта ≤ 30 минут
- Если данные недоступны → написать "NO DATA: причина" и перейти к следующему
