# Оптимизация: ограничение суммарной маржи + maxDD ≤ 5%

## Задача

Модифицировать `scripts/capital_growth_sim.py`:
1. Добавить в `simulate()` параметр `total_margin_limit` — максимальная доля капитала, которая может быть заблокирована в ГО суммарно по всем открытым позициям
2. Переоптимизировать grid search с новым параметром
3. Найти комбинации, где max drawdown не превышает 5% (жёсткое ограничение)
4. Среди них выбрать те, где equity растёт по экспоненте (максимальный финальный капитал)

## Текущая проблема

Сейчас нет ограничения на суммарное ГО. `margin_usage=0.02` на сделку × `max_concurrent=5` = 10% капитала в марже. Но если сигналы приходят плотно, может быть больше — ГО блокируется и не освобождается до закрытия позиции.

Результат сейчас: maxDD=38.7%, потому что капитал слишком глубоко в рынке в неудачные серии.

## Изменения в simulate()

Добавить параметр `total_margin_limit: float` (доля капитала, макс суммарное ГО, например 0.05 = 5%).

В функцию simulate() перед блокировкой ГО добавить проверку:

```python
# ── Проверка лимита суммарной маржи ──
total_locked = sum(p['locked_go'] for p in active.values())
if total_locked + locked_go > total_cap * total_margin_limit:
    continue  # превышен лимит суммарной маржи
```

Порядок проверок в simulate():
1. Drawdown limit (DD > max_dd_limit → стоп)
2. Закрытие старых позиций по этому тикеру
3. **Лимит concurrent** (`len(active) >= max_concurrent`)
4. **Лимит суммарной маржи** (новый)
5. Расчёт контрактов
6. Блокировка ГО и открытие

## Параметры grid search

```python
param_grid = {
    'margin_usage': [0.005, 0.01, 0.015, 0.02, 0.03, 0.05],  # доля капитала на 1 сделку
    'max_concurrent': [2, 3, 5, 8, 10],                        # макс параллельных
    'total_margin_limit': [0.03, 0.05, 0.08, 0.10, 0.15],     # макс суммарное ГО
    'max_dd_limit': [0.05, 0.08, 0.10],                        # стоп при просадке
    'stop_loss_pct': [0.005, 0.01, 0.015, 0.02],              # стоп-лосс на сделку
}
```

Всего: 6 × 5 × 5 × 3 × 4 = 1800 комбинаций на фолд.

**ВАЖНО**: total_margin_limit должен быть реалистичным. Если margin_usage=0.05 и total_margin_limit=0.05 — может открыться только 1 позиция. Если margin_usage=0.01 и total_margin_limit=0.05 — до 5 позиций при одинаковом ГО.

## Новая scoring функция

1. Запустить grid search
2. Для каждой комбинации посчитать max_drawdown на train
3. **Отфильтровать** комбинации с max_drawdown > 5%
4. Среди оставшихся выбрать по Calmar ratio (return/maxDD)
5. Если ни одна не прошла фильтр 5% — ослабить до 8%
