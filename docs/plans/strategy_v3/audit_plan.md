# Аудит результатов Portfolio Optimizer

## Подозрительные результаты

### 1. +34,526% — явный баг
Среди результатов есть `ret=34526.3% DD=55.57% Calmar=621.33`. Это невозможно при 935 сделках с mu=0.15. 
Проверить: что пошло не так в simulate_adaptive_portfolio() на комбинации mu=0.15 mc=5 tm=0.3 sl=0.02.

### 2. +692% — правдоподобно или нет?
100K → 791K за 1 год. 1052 сделки, mu=0.20, sl=0.02.
Проверить:
- Нет ли look-ahead bias в portfolio.py (используются ли данные из будущего?)
- Правильно ли idx сопоставляется с сигналом (не путаются ли тикеры?)
- Правильно ли считается PnL (не перепутаны LONG/SHORT)?

### 3. Portfolio vs FIFO: почему разрыв 17x?
FIFO даёт +39.5%, Portfolio +691.6% на тех же сигналах.
Проверить:
- Корректно ли работает priority eviction (вытеснение)?
- Не пропадают ли сигналы из-за бага в correlation check?
- Правильно ли считаются контракты при priority weights?

## Проверки

### A. Look-ahead bias
Для каждой функции в portfolio.py проверить: все ли rolling windows (z-score, ADX, ATR, SMA) используют только данные ДО текущего бара. Никаких данных из будущего.

### B. Signal idx integrity
Сигнал имеет idx = индекс в data. Проверить: совпадает ли data[idx] по времени с временем сигнала? Если нет — idx битый.

### C. PnL sanity check
Для 5 случайных сделок из portfolio sweep: пересчитать PnL вручную по формуле из tracker.py. Сравнить с тем, что выдала симуляция.

### D. Equity curve
Нет ли скачков капитала >50% за одну сделку? Если есть — баг.

## Вывод
Что из результатов реально, что — баг. Какие цифры можно доверять.
