═══ 1. ROOT CAUSE: Look-ahead bias in position exit on eviction ═══════════════

File: trading_bot/portfolio.py, lines 225-242

When a position is evicted (max_concurrent reached + higher-priority signal),
the PnL is computed using pos['exit_price'] — which is the signal's PRE-COMPUTED
exit price at the horizon (12 bars ahead).

But the eviction occurs at the TIME of the incoming signal, which may be BEFORE
the evicted position's horizon has elapsed. The exit price comes from a future
bar that hasn't been reached in simulation time.

Example:
  1. Signal for ticker A at bar 100: exit_price = close of bar 112 (horizon=12)
  2. Position A is opened at bar 100 with exit_price = close[112]
  3. Signal for ticker B arrives at bar 103
  4. If portfolio is full and B has higher priority, A is evicted
  5. PnL uses close[112] — but simulation time is only at bar 103!
  6. close[112] is 9 bars in the future → LOOK-AHEAD BIAS

The same bias exists in the FIFO version (portfolio_sweep.py:168-173) when a new
signal for the SAME ticker closes an old position. However, Portfolio is affected
DISPROPORTIONATELY because:
  - FIFO only closes same-ticker positions (rare, ~1 overlap per ticker)
  - Portfolio evicts ANY lower-priority ticker whenever at max_concurrent
  - With 7093 signals across 47 tickers, evictions are FREQUENT

═══ 9. RECOMMENDATION ═════════════════════════════════════════════════════════

  To fix, the exit price for an evicted position must be the exit price at
  the CURRENT bar (or entry + stop-loss if applicable), NOT the pre-computed
  horizon exit price. The position should be marked to market at eviction time.

  Two approaches:
  1. (Simple) When evicting, close at the current signal's exit price rather
     than the stored exit_price. This is a rough approximation.
  2. (Correct) Store the entry bar index and horizon, and look up the actual
     close price at the eviction time. This requires the full OHLCV data to
     be available in simulate_adaptive_portfolio().

  The same fix should be applied to the FIFO version for same-ticker overlaps.
