# Глубокий анализ Volume × OI: кандидаты NR, CC, MG, VB, PD, NG, IB, TN

## Противоречие
Объёмные всплески (volume z-score > 3σ) не дают предсказательной силы сами по себе — корреляция ∼0. Но когда классифицируешь их по OI-составу (yur_accumulation vs fiz_panic), некоторые тикеры показывают WR 55-59%.

## ИКР (Идеальный Конечный Результат)
Система, которая по одному бару объёмного всплеска определяет: будет прибыльным вход против юрлиц или нет — с точностью > 60% WR и avg return > 0.1%.

## Гипотезы (5 направлений)

### 1. Кросс-валидация по годам
Проверить: работает ли yur_accumulation на NG/CC/NR/MG стабильно по годам или результат одной вспышки.
- Разбить 2 года на 4 полугодовых периода
- Для каждого: WR + avg return yur_accum
- Фильтр: WR > 50% во всех периодах

### 2. Адаптивный z-score
Текущий z-score(20) — фиксированное окно. Для NR (мало данных) может быть шум.
- Попробовать z-score(40), z-score(60)
- Попробовать adaptive: std по EWMA, а не равновесному rolling

### 3. Контекст перед спайком
Не «что на spike-баре», а «как OI вёл себя 5-10 баров ДО».
- fiz_share рос или падал перед спайком?
- Цена была на экстремуме канала (ATR от EMA)?
- Был ли предыдущий кластер спайков?

### 4. Тикер-специфичный анализ
Каждый кандидат — свой рыночный режим:
- **NG** (Natural Gas) — товарный, высокая вола
- **CC** (Copper) — промышленный металл
- **NR** (Norilsk Nickel) — редкий, мало баров
- **MG** (Magnesium) — малоликвид
- **VB** (VTB) — банковский сектор
- **PD** (Palladium/Polyus) — золотодобыча или металл
- **IB** — ?

Проверить: какой у них fiz_share, какой avg volume, как спайки распределены по времени.

### 5. Оптимизация параметров
- Порог vol_z: 2.5, 3.0, 3.5, 4.0
- Порог yur_z: 1.0, 1.5, 2.0
- Horizon: 3, 6, 12 баров

## План

### Фаза 1 — Глубокий дайв по каждому кандидату
Для каждого из NR, CC, MG, VB, PD, NG, IB, TN:
1. Полные статы: бары, медианный объём, fiz_share, сессионное распределение
2. WR yur_accum по полугодиям (4 периода)
3. Лучшая комбинация порогов (vol_z, yur_z, horizon)

### Фаза 2 — Что общего у работающих тикеров
Построить профиль «идеального тикера» для Volume×OI стратегии.

### Фаза 3 — Walk-forward бэктест
Для тикеров, прошедших Фазу 1 (50% WR во всех периодах): портфельная симуляция с walk-forward, bar-level MTM.
